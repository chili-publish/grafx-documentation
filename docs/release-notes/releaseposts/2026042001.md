# GraFx Studio and Platform: Subject types, Brand Kit sync, and improvements

![GraFx Studio icon](/assets/icon-GraFx-Studio.svg){.rn_icon}
![CHILI GraFx icon](/assets/icon-CHILI-GraFx.svg){.rn_icon}

## ✨ New & Improved

### Subject types for Smart Crop

You can now assign a **subject type** to an asset — a classification that tells GraFx Studio what kind of subject the image contains (for example: person, product, or logo). When Smart Crop mode is active, the subject type is used to influence how the crop is applied, giving you more predictable results across different image categories.

Subject types are also visible in the **media details view**, so you can review and manage the classification directly alongside the asset's other metadata.

### Subject type management in the Platform

Subject types are managed centrally in CHILI GraFx. A dedicated **Subject types** page is now available in the platform, where environment admins can define and maintain the list of subject types available across their environment.

### Brand Kit sync

Brand Kits in GraFx Studio can now be **synced** to the source Brand Kit. When a Brand Kit is updated in GraFx Brand Kits, templates using it can reflect those changes without having to re-import manually. This keeps designs consistent with the latest brand definitions as they evolve.

## 🔧 Improvements

### Clearer feedback for constrain proportions with rotation and autogrow

When a text frame has both **rotation** and **autogrow** enabled, constraining proportions can produce unexpected results. An informational message is now shown in this situation, making the interaction between these settings more transparent.

### Output page ID for server rendering

An output page ID can now be passed as part of the image options during server-side rendering. This is primarily relevant for integrations that need to target a specific page during output generation.

## 🛠️ Fixes

- **PDF overprint** — embedding PDF assets that contain overprint statements no longer strips the overprint from the final PDF output.
- **Image variable metadata mappings** — metadata mappings on image variables can now be updated after being set for the first time.
