# Environment API Multi-File PDF Output

## ✨ New & Improved

- **Added** `maxRecordsPerFile` control in the Environment API — split large batch jobs into multiple PDF files.
- **Enabled** parallel rendering for PDF outputs, boosting reliability and throughput.
See your tenant Swagger for more information
https://[yourtenant]/grafx/swagger/index.html#/Output/post_api_v1_environment__environment__output_pdf

## 🛠️ What's Fixed

- **Fixed** edge cases in high-volume PDF rendering where outputs could stall or fail near record limits.
- **Updated** internal handling for large data sources to stabilize rendering across distributed workers.

## Outcome

Large batch exports in GraFx can now be split and processed efficiently with user-defined file limits and parallel rendering support in the Environment API. This unlocks stable processing of high-record data sources and smoother integration behavior.
