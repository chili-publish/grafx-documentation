# CHILI GraFx: Brand Kit Themes

![GraFx Brand Kits icon](/assets/icon-GraFx-Brandkits.svg){.rn_icon}

## ✨ New & Improved

### Themes: one Brand Kit, multiple variations

A Brand Kit is the single source of truth for a visual identity — its colors, fonts, media, and text styles are defined once and reused across Design Systems. Until now, a variation of that identity (a sub-brand, a print-specific palette, a regional style) meant maintaining a separate Brand Kit.

With **themes**, one Brand Kit now carries those variations itself. Every Brand Kit has a **default theme** holding the complete set of elements. Additional themes inherit everything from the default and store only what you override — change a color in a theme, and just that color diverges; update the default theme, and every theme that doesn't override the element picks up the change automatically.

Typical uses: a sub-brand or flavor overriding the accent color and logo, or a **Print** theme swapping RGB colors for their CMYK counterparts. See [Use cases](/GraFx-Brand-Kits/concepts/use-cases/) for more.

### Managing themes in GraFx Brand Kits

The Brand Kit detail page has a new theme bar with a back button to all Brand Kits, a dropdown listing all themes (the default theme is marked **(Default)** and listed first), and a manage icon that opens the **Brand Kit themes** panel. There you can create, rename, and delete themes; the default theme can only be renamed.

Editing a theme works exactly like editing a Brand Kit: select the theme in the dropdown and make your changes — they are saved automatically as overrides on that theme. Deleting the theme you are viewing returns you to the default theme.

### Use themes in GraFx Studio

The Brand Kit panel in the Studio workspace now has a **theme selector**. When you open a template, the default theme is selected; switching to another theme updates every brand-bound value in the template — colors, text styles, and media — on the canvas and in the property panels.

The structure of a Brand Kit is managed on the default theme: adding, renaming, removing, or duplicating elements is disabled on non-default themes, and an info notice in the panel explains this. On a non-default theme you work with the values, not the structure.

Brand Kit sync is theme-aware: a sync applies to the selected theme and restores its settings to the source Brand Kit's state.

### Switch themes via Actions and the SDK

Theme switching is also available programmatically, so a template can change its brand variation in response to what the end user does — for example, switching to the Print theme when the layout changes to a print layout. Two new Actions helper functions are available:

```
getActiveThemeName(): string
```

```
switchTheme(name: string): void
```

Integrations can do the same through the Studio SDK's brand kit theme methods.

!!! info "Fonts and themes"
    The Brand Kit's font list is shared across all themes — by design, a theme does not add or remove fonts. Styles, however, can be overridden per theme to use a different font from the shared list.

### GraFx Studio improvements

This Studio release also brings a set of smaller improvements. Heavy media assets no longer fail prematurely: Studio now waits as long as GraFx Media may need (up to 10 minutes) to process an asset before giving up. Font previews got an additional fallback, so previews render in more cases. Undo and redo in text editing behave more predictably. On the asset details view, when a custom subject type has no settings of its own, the settings from the default subject type are now used. And long layout and variable names are now easier to read in their panels.

## 🐛 Bug Fixes

This release fixes the canvas rendering in the space between the left panels, text selection in input fields using various mouse clicks, validation in particular input fields, and a case where text went into overflow with copyfitting and auto-grow enabled on multicolumn frames. PDF generation performance is also improved.
