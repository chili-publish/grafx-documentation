# CHILI GraFx Environment API: Source files preserved in print output

![CHILI GraFx icon](/assets/icon-CHILI-GraFx.svg){.rn_icon}

## 🛠️ Fixes

### JPG and PNG source files are kept for print output

When GraFx Studio generates a PDF, the output engine requests each placed image at full resolution for print. For JPG and PNG assets, the [media connector documentation](/GraFx-Developers/connectors/media-connector/media-connector-fundamentals/#supported-formats) specifies that the original source file should be returned in its highest quality — not a re-processed copy.

Until now, JPG and PNG assets were re-encoded during print output instead of being passed through. That re-encoding could change the file — most visibly, a CMYK JPEG was converted to RGB, shifting colors in the final PDF. The Environment API now returns the original source file for JPG and PNG media during output requests, so the asset's original color space and quality are preserved. A CMYK JPEG placed in a template now stays CMYK all the way through to the printed PDF.
