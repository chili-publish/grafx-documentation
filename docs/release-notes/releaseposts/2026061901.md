# CHILI GraFx Environment API: Font preview fix and conditional requests

![CHILI GraFx icon](/assets/icon-CHILI-GraFx.svg){.rn_icon}

## ✨ New & Improved

### Faster repeat requests with ETags

The Environment API now supports conditional requests on read operations. Every `GET` and `HEAD` response includes an **ETag** — a short fingerprint of the returned content. On your next request for the same resource, pass that value back in an `If-None-Match` header: if nothing has changed, the API replies with `304 Not Modified` and no body, so your integration skips re-downloading data it already has. This makes frequently repeated reads faster and lighter on bandwidth, with no change to the data you receive when content *has* changed. Requests that modify data (`POST`, `PUT`, `DELETE`) are unaffected and always run as normal.

## 🐛 Bug Fixes

### Font previews render correctly for newly uploaded fonts

A font preview is the sample image GraFx generates for each font style so you can see the typeface at a glance. The square (400×400) preview for newly uploaded fonts was rendering its sample text on a single line instead of wrapping to fill the square. Previews now wrap correctly, and previews that were already affected have been refreshed — so both new and existing fonts display as intended.
