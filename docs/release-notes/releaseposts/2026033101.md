# GraFx Studio Update

![GraFx Studio icon](/assets/icon-GraFx-Studio.svg){.rn_icon}

## ✨ New & Improved

### Image crop constraint for end users

Template designers can now control whether end users are allowed to crop images within a frame. This is a per-frame setting — disabled by default.

When crop is enabled for a frame, end users see a crop icon alongside the existing rotation icon. Activating crop mode lets the user drag the image within the frame to adjust what's visible. The frame size and position stay fixed — only the image's position inside the frame changes.

**A note on fit modes:**

- For frames using Fit, Fill, or Smart crop: each image keeps its own individual crop when adjusted by the end user.
- For frames using Manual crop: there is no per-asset crop. End users adjust the shared crop area of the frame, which applies to all images in it. If you want end users to crop images individually per asset, do not use Manual crop fit mode on that frame.

### Connector shared header value

Connector developers can now set a header value that applies across all current and future instances of a connector sharing the same remote connector ID. This makes it easier to pass shared configuration — such as an API key or session token — once, without needing to set it per connector instance.

### Font size constraint enabled by default

When text editing constraints are turned on for a text frame, the font size constraint is now enabled by default. Previously it was off, which could lead to unexpected results when the intent was to restrict all text editing properties.

## 🐛 Fixed

- **Templates with components now generate digital output correctly** — MP4, GIF, PNG, and JPG output previously failed with an internal error when a template contained components. This is resolved.
- **Missing font no longer crashes output** — if a font referenced in a template is not available, output generation now continues with a fallback rather than failing entirely.
- **Images can now be assigned to frames with Smart crop** — a regression had prevented new images from being assigned to frames that had Smart crop mode active. This is fixed.
- **Image crop in Manual crop mode now saves correctly** — crop settings applied in project (Run) mode were not being saved for images using Manual crop fit mode. This is now resolved.
- **Datasource modal error state** — the modal now correctly shows an empty or error state rather than displaying incorrectly when the datasource returns no data.
- **Text constraints rotation label** — a typo in the rotation info message inside the text constraints settings has been corrected.
