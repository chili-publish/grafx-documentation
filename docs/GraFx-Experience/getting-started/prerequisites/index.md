# Prerequisites for GraFx Experience

## Access & licensing

- Your organization has an active **CHILI GraFx** subscription that includes **GraFx Experience**.
- Your user account has been invited to the Experience environment.
- You have the correct **role** and **group memberships** to see and use Design Systems.

## User authentication

- You have a valid **local account** or federated access via **Single Sign-On (SSO)**.
- If using SSO, the identity provider is configured and linked to the platform.
- Your user is mapped to the correct groups so permissions are granted.

## Available Design Systems

- At least one **Design System** is published and visible in GraFx Experience.
- Design Systems include defined variables, business logic, and guardrails.
- You know which Design Systems you will use.

## Asset & brand setup

- Approved media assets (images, logos, icons, etc.) are available in **GraFx Media** or via a connected DAM through the **Connector Framework**.
- Brand identity elements (colors, fonts, styles) are configured in **GraFx Brand Kits**.
- Users who need media upload/edit rights belong to groups with those permissions.

## Browser support

- See [Browser Support](/CHILI-GraFx/browser-support/)

## Optional integrations

- External data sources (e.g., PIM, CRM, CDP) are connected if needed.
- Connectors and API endpoints have been configured by your technical team.
- Any identity provider attributes needed for SSO group mapping are set up.
